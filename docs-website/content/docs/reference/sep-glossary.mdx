---
title: SEP Glossary
description: Stellar Ecosystem Proposals used in NordStern
---

The Stellar Ecosystem Proposals (SEPs) define how different participants on the network (like wallets and anchors) interact with each other. NordStern leverages several key SEPs.

## SEP-1: Stellar Info File
A standard `stellar.toml` file hosted by the anchor. It acts as a discovery mechanism, telling wallets where to find the anchor's API endpoints (like KYC or deposit servers) and providing details about the issuer and the assets.

## SEP-10: Stellar Web Authentication
A standard for authenticating users. It uses Stellar transactions as challenge-response mechanisms, allowing wallets to log in to the anchor's server by proving they own a specific Stellar private key.

## SEP-12: KYC API
A standard API for an anchor to collect Know Your Customer (KYC) information from users. It allows wallets to upload customer data (names, IDs, photos) securely to the anchor.

## SEP-24: Hosted Deposit and Withdrawal
The core of NordStern's offering. It defines an interactive flow where a wallet opens a webview provided by the anchor. The user completes the fiat-in (deposit) or fiat-out (withdrawal) process within this webview, and the anchor mints or burns the corresponding tokens on the Stellar network.
