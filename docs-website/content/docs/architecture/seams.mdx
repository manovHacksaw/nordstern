title: The Seams (Swappable Adapters)
description: Why everything external is modular in NordStern
---

## Design Philosophy

**Everything external is a swappable adapter behind an interface.**

Because the legal model surrounding crypto and fiat in India is unsettled (who holds the fiat, under what license, whose KYC applies, etc.), we architect for optionality.

KYC providers, fiat-in (UPI/collections), fiat-out (payout PSPs), and banking/custody are all built as **seams**. 

## Interfaces over Implementations

We program to an interface with a **mock implementation as the default**. Real vendor logic is placed behind this interface, ensuring that a specific vendor (like Cashfree, RazorpayX, or HyperVerge) never leaks into the core flow logic.

### Key Adapters

- **`KycProvider`**: For SEP-12 KYC. Currently mocked to always accept, but built to swap in HyperVerge or Signzy.
- **`DepositProvider`**: For fiat-in (UPI collection).
- **`PayoutProvider`**: For fiat-out. Cashfree / RazorpayX integrate here.

By maintaining these seams, when the time comes to finalize the legal banking partnership, the core system does not require a rewrite.
