## Overview

The `control-plane` service is the seed of our future SaaS platform. Currently, it acts as a lightweight management layer built with TypeScript/Express and PostgreSQL (`controldb`), utilizing JWT + bcrypt for authentication.

## Purpose

While the MVP is focused on a single anchor we operate, it is structured so it can eventually generalize to a multi-tenant platform. The control plane handles:

1. **Tenant Authentication:** Managing logins for the operator dashboard.
2. **Keypair Provisioning:** Generating and assigning Stellar keypairs for the anchor operations.

## Multi-tenant Philosophy

> [!IMPORTANT]
> **One anchor now, multi-anchor later — but leave the seam.** 

We build the single anchor as if tenant identity will eventually scope every asset, keypair, and transaction. We do not build the full multi-tenant control surface yet, nor do we turn "add an anchor" into an instant public signup (we onboard anchors by hand for now). 

However, we never design anything that would make adding a tenant dimension a rewrite later on.
